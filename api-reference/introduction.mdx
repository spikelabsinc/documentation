## Base URL

All API requests should be made to:

```
https://api.spike.dev/v1
```

For sandbox/testing:

```
https://api.spike.dev/v1-sandbox
```

## Authentication

The Spike API uses Bearer token authentication. Include your API key in the `Authorization` header:

```bash
curl https://api.spike.dev/v1/personalities \
  -H "Authorization: Bearer sk_live_your_api_key"
```

API keys can be created and managed in the [Spike Dashboard](https://dashboard.spike.dev).

| Key Prefix | Environment |
|------------|-------------|
| `sk_live_` | Production |
| `sk_test_` | Sandbox |

## Response Format

All responses are JSON. Successful responses include the requested data:

```json
{
  "id": "personality_abc123",
  "object": "personality",
  "name": "Luv",
  ...
}
```

List endpoints return paginated results:

```json
{
  "data": [...],
  "has_more": true,
  "next_cursor": "eyJ..."
}
```

## Errors

Errors return an appropriate HTTP status code and a JSON body:

```json
{
  "error": {
    "code": "invalid_request",
    "message": "The 'name' field is required",
    "param": "name"
  }
}
```

| Status Code | Description |
|-------------|-------------|
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid or missing API key |
| 404 | Not Found - Resource doesn't exist |
| 409 | Conflict - Resource already exists |
| 429 | Too Many Requests - Rate limited |
| 500 | Server Error - Something went wrong |

## Rate Limits

API requests are rate limited per API key:

| Plan | Requests/min |
|------|--------------|
| Free | 60 |
| Pro | 600 |
| Enterprise | Custom |

Rate limit headers are included in responses:

```
X-RateLimit-Limit: 600
X-RateLimit-Remaining: 599
X-RateLimit-Reset: 1704067200
```
